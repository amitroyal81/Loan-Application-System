🔄 WORKFLOW EXAMPLE

1. User submits via Streamlit UI
   → Fill form, click submit

2. FastAPI receives application
   → POST /api/v1/apply
   → Returns acknowledgment with case ID

3. Background task calls orchestrator
   → process_loan_application()

4. Orchestrator coordinates agents (sequentially)
   State 1: Applicant Profile Analysis
   State 2: Financial Risk Analysis
   State 3: Loan Decision Synthesis
   State 4: Compliance & Actions

5. Each agent uses Claude Sonnet 4.6 LLM
   → Provides reasoning and confidence scores

6. Results synthesized into final decision
   → Classification: Approved/Rejected/Manual Review
   → Risk Score: 0-100
   → Confidence: 0-100%
   → Key Factors & Explanation

7. Results stored with full audit trail
   → Case ID, timestamp, all agent outputs

8. UI polls for decision
   → GET /api/v1/status/{applicant_id}
   → Displays results with visualizations
